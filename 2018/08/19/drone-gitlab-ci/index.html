3. 配置Gtilab

  • 个人头像–>settings–>Applications

    • Redirect URI对应docker-compose.yml中的{DRONE_HOST}/authorize

  • 保存：

    • Application ID对应dcoker-compose.yml中的DRONE_GITLAB_CLIENT
    • Secret对应docker-compose.yml中的DRONE_GITLAB_SECRET

  • Admin Area（小扳手标识）–>Settings–>Network–>Outbound requests–>勾选Allow requests to the local network from hooks and services

4. 使用

  • Drone平台：http://{IP:port}
  • 第一次登录会先跳转至Gitlab初始化root账户的密码修改，其后通过root/password的方式登录，跳转到Drone平台
  • Drone平台会自动同步Gitlab上的项目，并在drone平台上予以激活，便可通过代码提交，触发drone pipeline
  • 在项目根目录编写.drone.yml

.drone.yml

pipeline:
  build:        # step name
    image: node # use docker image
    commands:
      - npm install     # install node project vendor
      - npm run build   # build static files

5. 配置nginx反向代理

总是用IP:PORT的方式如果觉得不方便，可以设置下nginx反向代理，但此时需要重新添加Gitlab中的application以及docker-compose.yml中的key，因为跳转会失败。

vim /etc/nginx/conf.d/drone.conf

map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}
server {
    listen 80;
    # 此处替换为自己的ip
    server_name IP-Addr;
    set $drone_port 3800;
    location / {
        proxy_set_header X-Forwarded-For $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Host $http_host;
        proxy_pass http://127.0.0.1:$drone_port;
        proxy_redirect off;
        proxy_http_version 1.1;
        proxy_buffering off;
        chunked_transfer_encoding off;
    }
    location ~* /ws {
        proxy_pass http://127.0.0.1:$drone_port;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400;
        proxy_set_header X-Forwarded-For $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Host $http_host;
    }
}
